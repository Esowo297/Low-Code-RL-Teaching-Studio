## 1. 作业模板被写成可创建、发布、入库，但代码当前只提供内置模板目录

- 论文位置：`2.4.4`、`3.1.1`、`3.4.1`、`4.1.5`、`5.2.5`、`5.4.2`、`5.4.3`
- 论文表述要点：
  - 教师可整理、保存、发布作业模板。
  - 作业模板会写入数据层，学生进入页面时再读取回填。
- 代码实际情况：
  - 后端只有 `GET /api/assignments`，直接返回内置作业模板目录。
  - 没有作业模板的新增、修改、删除、发布接口。
  - SQLite 中也没有作业模板数据表。
- 代码依据：
  - `backend/app/api/routes.py`：`/api/assignments` 只调用 `build_assignment_catalog()`
  - `backend/app/services/assignments.py`：作业模板以代码常量方式构造
  - `backend/app/repositories/experiment_store.py`：数据库只创建 `experiments` 与 `benchmarks` 两张表
- 建议收口表述：
  - 当前原型提供“内置作业模板目录”，支持学生加载默认任务配置，但不支持教师在界面中创建、发布和持久化作业模板。

- 代码依据：

  - `backend/app/services/assignments.py`：模板来自内置目录

- 建议收口表述：
  - 当前 SQLite 主要承担“实验结果 + 教师自定义基准”的持久化，不包含作业模板表。

## 5. WebSocket 完成事件和控制语义与论文描述不完全一致

- 论文位置：`4.5.2`、`4.5.3`
- 论文表述要点：
  - 训练正常结束后返回摘要指标、最终路径和保存记录编号。
  - 控制指令根据当前任务编号修改任务状态。
- 代码实际情况：

- 代码依据：

  - `backend/app/api/routes.py`：`pause` / `resume` / `cancel` 直接调用当前连接的 `controller`
- 建议收口表述：
  - WebSocket 的完成事件返回完整实验结果对象；暂停、继续、取消是针对当前训练连接的控制，而不是根据外部任务编号路由到任意任务。

## 6. 指标曲线的语义被写成统一“探索率 / 误差”，但代码里不同算法含义不同

- 论文位置：摘要相关描述、`2.3.3`、`4.4.2`、`4.5.2`、`5.3`
- 论文表述要点：
  - 训练过程中统一返回奖励、探索率、误差或损失。
  - 某些地方容易被理解为不同算法会各自显示完全不同的图表。
- 代码实际情况：
  - 前端统一显示三张曲线图：`reward`、`epsilon`、`td_error`。
  - 但这两个字段在不同算法下语义不同：
    - Q-Learning / SARSA：`epsilon` 为探索率，`td_error` 为平均 TD 误差
    - DQN：`epsilon` 为探索率，`td_error` 为平均损失
    - REINFORCE：`epsilon` 为归一化策略熵，`td_error` 为策略损失
- 代码依据：
  - `frontend/src/App.vue`：统一渲染三张 `MetricChart`
  - `backend/app/rl/algorithms/q_learning.py`
  - `backend/app/rl/algorithms/sarsa.py`
  - `backend/app/rl/algorithms/dqn.py`
  - `backend/app/rl/algorithms/reinforce.py`
- 建议收口表述：
  - 平台复用统一指标结构承载不同算法的训练过程，其中 `epsilon` 与 `td_error` 在不同算法下具有不同但可解释的教学语义。

  - `backend/app/services/analytics.py`
  - `backend/app/schemas/experiment.py`：`ClassroomAnalyticsResponse`
- 建议收口表述：
  - 当前课堂分析定位为“已保存实验结果的聚合统计接口”，而不是任意条件检索或分析明细接口。

## 9. 报告导出被写成依赖已保存记录，但代码允许直接基于当前结果导出

- 论文位置：`4.1.6`、`5.2.4`、`5.4.2`、`5.4.3`
- 论文表述要点：
  - 报告导出继续读取已保存配置、指标和路径样本。
- 代码实际情况：
  - 报告导出接口接收的是 `ExperimentReportRequest`，其中直接携带当前 `result` 对象。
  - 如果传入 `benchmark_id`，系统会额外读取教师基准；但报告导出本身不强制依赖历史记录已入库。
- 代码依据：
  - `backend/app/api/routes.py`：`POST /api/reports/render`
  - `backend/app/services/reporting.py`：直接使用 `request.result`
  - `frontend/src/api.ts`、`frontend/src/App.vue`：前端可直接导出当前实验结果
- 建议收口表述：
  - 报告导出以“当前实验结果对象”为主输入，可选结合教师基准信息，不必限定为“只能从已保存历史中导出”。
